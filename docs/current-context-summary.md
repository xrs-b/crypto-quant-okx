## 当前仓位控制核心
- 杠杆基准：`3x`
- 原则：**风险预算优先**，不是见信号就固定仓位开单。
- 同向加仓：**默认禁止**
- 单笔保证金目标：**围绕 8% 动态调整**
- 单笔动态范围：**4% ~ 10%**
- 总保证金上限：**30%**
- 逻辑重点：先看当前总风险占用 / 单币种占用，再决定是否允许开仓，以及本次能开多大。

## 当前已知运行结论
- `❌ 开仓执行失败` 曾出现误报：实际不一定是交易所拒单，可能是本地开仓后处理异常。
- `⚠️ 持仓对账异常` 的典型场景：交易所 / 本地 position 有仓，但本地 `open_trades` 缺失，导致每周期重复提示。
- `The read operation timed out` 更像交易所接口或网络瞬时超时，不应一概按致命交易失败理解。

## 2026-03-25 已做修复
- 开仓后本地补记兜底：若交易所已成交但本地记录失败，尝试从交易所持仓恢复 `trade + position`。
- 对账自愈：发现 `exchange_missing_open_trade` 时，自动补建 open trade。
- 告警降噪：
  - 对账异常独立去重，避免每周期洗版
  - timeout 类错误降级，不再一律按最严重告警呈现
- 机器人重复实例问题已处理：此前有两个 daemon 同时运行导致每轮多出 `⏭️ 机器人周期跳过`；现已收敛为单实例运行。
