## 本批已修复问题

### 1. 下单数量未尊重交易所上限
- **现象**：频繁触发 `51202 Market order amount exceeds the maximum amount`
- **修复**：`normalize_contract_amount()` 同时尊重 `min/max amount`
- **提交**：`c7335f7`

### 2. 交易所无仓，但本地 open trade 仍挂 open
- **现象**：本地 open trades / positions 与交易所状态长期不一致
- **修复**：对账时自动收口脏 open trades，并写入 `自动收口` notes
- **提交**：`fc18fa2`

### 3. 平仓遇到 51169 时只会重试，未做自愈
- **现象**：`you don't have any positions in this direction`
- **修复**：遇到 `51169` 后主动查询交易所真实持仓，若确认无仓则自动收口本地持仓/交易
- **提交**：`6299cd7`

### 4. Trailing stop 锚点未持续跟踪
- **现象**：
  - long 未持续记录最高价
  - short 未持续记录最低价
  - 导致追踪止盈/止损判断失真
- **修复**：
  - 开仓后初始化锚点
  - 后续检查持续更新 high/low anchor
  - 平仓或自动收口后清理缓存
- **提交**：`47beb43`

## 当前剩余已知风险 / 候选问题

以下问题仍值得后续继续处理，但优先级已低于本轮已修项：

### A. 开仓链路仍可能先撞一次 51202 再缩量重试
虽然已加 max amount 限制，但如果交易所真实限制与 market metadata 存在偏差，仍可能在极端情况下继续靠 fallback/retry。

**建议后续方向：**
- 增加更精细的 amount 预检/缓存
- 将失败码统计接入 dashboard 风险面板

### B. 风控与执行摘要仍偏“结果汇总”，未做到逐阶段根因归档
当前已经有 runtime context，但离“每轮完整诊断报告”仲有距离。

**建议后续方向：**
- 为每轮 run 增加 structured audit summary
- 将 signal → decision → trade → close 错误串成同一条 trace

### C. memory 检索仍未修复
这不属于交易链路 bug，但会影响后续长链协作效率。
