# OKX 量化系统使用说明与调参指南（老细版）

> 目标：用最少干预，管理一套会自动开单 / 平单 / 风控 / 解释信号既 OKX 量化系统。

---

## 1. 呢套系统系做咩用？

呢套系统唔系纯粹一个“下单脚本”，而系一套**低干预、可监督、可解释、可回滚**既自动交易系统。

你可以理解成：

- **机器人负责执行**：按策略、风控、市场过滤自动开平仓
- **仪表台负责解释**：俾你睇点解通过 / 点解唔通过 / 最近做过乜
- **你负责监督**：唔需要盯每一单，但要定期巡检、理解系统行为、必要时调参

### 你日常唔需要做既事
- 唔需要手动逐单盯盘
- 唔需要每次有信号都介入
- 唔需要一发现输单就即时乱改参数

### 你日常需要做既事
- 睇 dashboard
- 睇风控、信号、过滤原因
- 睇系统建议同 override 生效情况
- 有证据时再调参

## 2. 你依家最应该点用呢套系统？

现阶段最适合你既模式系：

## **低干预监督模式**

即系：
1. 系统自动运行
2. 你主要睇数据同解释
3. 真系有证据显示参数唔合适时，先做 override 调整
4. 调整后继续观察，而唔系即日连环改多个参数

## 3. Dashboard 点睇？

如果你唔想被太多功能吓窒，建议先专注以下几个区块。

### A. 总览 / 风险
先睇：
- 当前持仓
- 当前仓位占用
- 今日信号数 / 执行率
- 有冇逼近最大风险暴露

### B. 信号记录
重点睇：
- 最近有咩信号
- 信号类型（buy / sell / hold）
- 强度几多
- 触发咗边啲策略
- 点解被过滤

### C. 按币种拆解观望 / 过滤
重点睇：
- 边只币长期观望
- 边只币长期被某类规则挡住
- 边只币已经开始有执行样本

### D. 过滤规则有效性
重点睇：
- 某条规则整体系帮你避险多
- 定系经常错失机会多

### E. 按币种参数建议器
重点睇：
- 建议放宽咩参数
- 建议保持咩参数
- 建议灰度测试边条规则

### F. override 区块
重点睇：
- 当前已生效既 override
- 最新生成既 draft
- backup / rollback
- 最近 apply / rollback 历史

## 4. 日常巡检流程（5 分钟版）

建议你平时按以下顺序巡检：

### 第一步：睇风险
问自己：
- 当前持仓多唔多？
- 仓位占用高唔高？
- 系统有冇异常暴露？

### 第二步：睇信号
问自己：
- 今日有冇信号？
- 信号大部分系通过、观望、定被过滤？
- 最近过滤原因集中喺边度？

### 第三步：睇按币种拆解
问自己：
- 系咪某只币长期卡住同一种问题？
- 边只币已经开始出现真实执行样本？

### 第四步：睇规则有效性 / 参数建议
问自己：
- 某条过滤规则整体帮紧手定拖紧后腿？
- 某只币有冇持续被建议放宽 / 保持？

### 第五步：先决定要唔要调参
原则：
- **先睇证据，后改参数**
- **先小改，后观察**
- **先 override，后考虑全局参数**

## 5. 参数点分类理解？

你唔需要一次过学晒所有参数，只要先掌握以下 4 类。

### 5.1 仓位类
例如：
- `trading.position_size`
- `trading.max_exposure`
- `trading.max_position_per_symbol`

#### 作用
控制你每次开几大、总共最多开几多、单币最多占几多。

#### 什么时候考虑调？
- 系统太激进
- 同时持仓太多
- 回撤超出你舒服范围

#### 调整影响
- 调低：更稳、更保守、更少单
- 调高：收益潜力大，但风险同步放大

#### 当前建议
- **保守优先**

---

### 5.2 进场门槛类
例如：
- `strategies.composite.min_strength`
- `strategies.composite.min_strategy_count`

#### 作用
决定信号要几强、要几多策略确认先准入场。

#### 什么时候考虑调？
- 太多信号因为“强度不足 / 触发策略不足”被挡住
- 后验分析显示经常错失机会

#### 调整影响
- 放宽：更多单，但更多噪音
- 收紧：更保守，但可能少单甚至错失行情

---

### 5.3 市场过滤类
例如：
- `market_filters.block_counter_trend`
- `market_filters.min_volatility`
- `market_filters.max_volatility`

#### 作用
决定系统会唔会因为逆势、低波动、高波动而拦截信号。

#### 什么时候考虑调？
- 某只币长期被同一类过滤挡住
- 而且后验统计显示经常错失机会

#### 调整影响
- 放宽：机会增多，但坏单都可能增多
- 收紧：稳阵啲，但容易错过突破段

#### 当前建议
- 呢类参数**优先用 symbol override 调整**
- 唔好急住全局一刀切

---

### 5.4 风控执行类
例如：
- `trading.stop_loss`
- `trading.take_profit`
- `trading.trailing_stop`
- `trading.cooldown_minutes`

#### 作用
控制止损、止盈、追踪止损、连开间隔。

#### 什么时候考虑调？
- 经常未跑出趋势就被扫止损
- 经常赢少少就过早止盈
- 冷却期太长 / 太短影响重复机会

#### 当前建议
- 如果历史样本未够，唔好太频繁改
- 先稳定仓位、门槛、过滤，再细调止损止盈

## 6. 什么时候应该调参？

### 正确理由
你可以考虑调参，如果出现以下情况：

1. 后验分析持续显示某规则错失机会偏多
2. 同一只币长期被同一个 filter_code 拦截
3. 参数建议器连续数轮都俾出同方向建议
4. 风险 / 回撤已超出你可接受范围
5. 你已经观察咗足够样本，而唔系只睇一两日

### 错误理由
以下情况唔应该即刻改：

1. 输咗一两单就改
2. 今日冇开单就急住放宽
3. 一见被过滤就觉得系统太保守
4. 未睇清楚过滤原因就乱调
5. 一次过改多个参数再话系统唔稳

## 7. override 系点用？

override 用途系：
## **俾单独某只币用唔同参数，而唔影响全局默认值**

例如：
- BTC 保持严格
- XRP 放宽 `min_strength`
- DOGE 放宽 `min_volatility`

### 相关能力
#### 1. 生成草案
系统会根据建议器，输出：
- 结构化 draft
- YAML draft 文本

#### 2. 应用草案
可一键将草案写入：
- `config/config.local.yaml`

#### 3. 生效可视化
可睇到：
- 当前 override 内容
- 实际生效值

#### 4. 回滚
可选 backup 回滚

#### 5. 审计历史
可追踪：
- apply 咗边次
- rollback 咗边次
- 涉及边只币
- 涉及几多个参数

---

## 8. 建议器动作点理解？

### 建议放宽（relax）
意思：
- 当前参数太严
- 后验统计显示经常挡走本来有机会既交易

### 保持当前（keep）
意思：
- 当前规则多数系帮你避险
- 唔应该因为少量错失就乱改

### 灰度测试（review）
意思：
- 呢类建议唔适合直接自动写入
- 应该小心地做局部试验
- 例如逆势过滤 `block_counter_trend`

---

## 9. 低干预模式下，你最应该遵守既 5 条规则

### 规则 1：先睇证据，再调参数
唔好因为感觉、焦虑、或者短期输赢而调。

### 规则 2：先做单币 override，后做全局调整
如果问题只喺 XRP 出现，就唔好全局改晒所有币。

### 规则 3：一次只改一类参数
例如先改 `min_strength`，唔好同一时间又改 `cooldown` 又改 `volatility`。

### 规则 4：改完要继续观察
调完唔系即刻下结论，要俾系统继续跑样本。

### 规则 5：风险类参数永远优先保守
仓位、暴露、止损、冷却期，宁愿稳阵先。

## 10. 你而家最建议既使用姿势

如果你问我“现阶段最稳阵点用”，我会建议：

### 默认姿势
- 全局参数保持保守
- 用 dashboard 做巡检
- 用建议器搵有问题既币种
- 用 draft → apply → 生效查看 → rollback 呢条链路做小步调整

### 唔建议既姿势
- 每日都手改参数
- 今日冇单就即刻放宽门槛
- 连续 apply 多份 override 唔观察
- 改咗之后又唔记得之前改过乜

你唔需要成为量化研究员，
